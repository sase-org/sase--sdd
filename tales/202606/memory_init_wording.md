# Plan: Update `sase memory init` Short-Term Memory Wording

## Goal

Change the generated short-term memory section in AGENTS-style files so `sase memory init` emits:

```text
The following memories contains core (always loaded) context:
```

instead of:

```text
The following memory contains core (always loaded) context:
```

Then run `sase memory init` so the managed agent instruction files in this repo are refreshed from the updated
generator.

## Scope

- Update only the AGENTS-style short-term memory section generated for AMD-managed instruction files.
- Keep minimal AGENTS rendering, memory README rendering, long-term memory text, inventory parsing, and unrelated
  generated-memory prose unchanged.
- Refresh checked-in provider instruction files generated by `sase memory init`, such as `AGENTS.md`, `CLAUDE.md`,
  `GEMINI.md`, `OPENCODE.md`, and `QWEN.md`, if the command updates them.
- Update focused tests that assert or exercise this exact generated wording.

## Implementation Approach

1. Change the managed AGENTS renderer in `src/sase/amd/_memory.py` to emit the requested sentence.
2. Update focused test coverage:
   - Add or adjust an assertion in the managed `sase memory init` test so it verifies the new sentence.
   - Update any existing fixture strings that hard-code the old sentence and are meant to represent current managed
     AGENTS output.
3. Run `sase memory init` from the repo root after the generator change to apply the new generated text to managed agent
   files.
4. Inspect the resulting diff to confirm it is limited to the intended source, test, plan/proposal, and generated
   instruction-file wording changes.
