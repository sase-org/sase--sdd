# Chat Transcript Metadata Description Lists Plan

## Context

SASE chat transcript files under `~/.sase/chats/` are generated by `save_chat_history` in `src/sase/history/chat.py`.
The current metadata header renders timestamp, model, and agent metadata on one physical line:

```markdown
**Timestamp:** 2026-05-28 09:16:41 EDT **MODEL:** codex/gpt-5.5 **AGENT:** reads.final-2
```

The desired format renders each metadata label and value as a compact Markdown description-style block:

```markdown
**Timestamp** 2026-05-28 09:16:41 EDT

**MODEL** codex/gpt-5.5

**AGENT** reads.final-2
```

The change should apply to newly saved transcripts. Existing transcript parsing should keep tolerating old files where
possible, because existing `~/.sase/chats/` content and test fixtures still use the old timestamp-line format.

## Proposed Scope

1. Update the transcript writer in `src/sase/history/chat.py`.
   - Replace the inline metadata suffix helper with a helper that returns labeled field blocks.
   - Always render timestamp as:
     ```markdown
     **Timestamp** <display timestamp>
     ```
   - Render optional `MODEL` and `AGENT` blocks only when their sanitized values are present.
   - Preserve the existing model formatting semantics: provider/model when both are present, avoid double-prefixing when
     the model already starts with `<provider>/`, and omit unknown fields.
   - Keep the prompt, previous-history, and extra-section ordering unchanged: metadata blocks first, then extra
     sections, previous conversation, prompt, and response.

2. Update linked-chat insertion in `src/sase/history/chat_links.py`.
   - The current insertion regex looks for a single `**Timestamp:**` line.
   - Teach it to insert after the new metadata block sequence when present.
   - Preserve fallback support for legacy `**Timestamp:** ...` transcripts so old files can still be linked.

3. Update focused tests and fixtures.
   - Adjust `tests/history/test_chat_save.py` expectations for the new block layout and ordering.
   - Add or update linked-chat tests to cover both new block headers and legacy timestamp-line headers.
   - Update minimal catalog and memory-episode fixtures where they are intended to represent current SASE transcript
     output, while preserving parser compatibility tests for old-style headers where useful.

4. Update user-facing documentation.
   - Revise the chat transcript file format example in `docs/llms.md`.
   - Leave historical SDD/tale/prompt artifacts untouched unless a live test or documented contract depends on them.

## Compatibility Notes

- `sase chats list` currently derives workflow, agent, timestamp, prompt, and response snippets from the H1, filename,
  and prompt/response headings, not from the timestamp line content. It should not need a parser change.
- `parse_chat_turns` and memory episode chat parsing key off prompt/response headings, so they should continue to work
  with both header formats.
- `append_links_to_chat` is the main consumer that structurally depends on the old timestamp line and needs explicit
  compatibility handling.

3. Inspect the final diff to confirm no memory files or unrelated historical artifacts were modified.
