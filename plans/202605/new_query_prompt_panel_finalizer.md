# Plan: Keep the agent's submitted prompt (with `# New Query` at the bottom) in the ACE prompt panel

## Problem / Product Context

In the `sase ace` TUI, the right-hand **AGENT PROMPT** panel for a forked agent (e.g. `@bp6.f1.f1.f1`) no longer shows
the `# New Query` section at the bottom of the prompt. The `# New Query` block is what `src/sase/xprompts/fork.yml`
appends when resuming a prior conversation, and for a fork it should be the _last_ thing in the agent's prompt (Previous
Conversation → `---` → `# New Query` → the user's new instruction).

Instead, the panel shows the user's new instruction buried in the middle of the prompt, followed by
`--- Work So Far ---` and `--- Commit Finalizer Pass N ---` sections.

## Root Cause (confirmed)

The displayed prompt is **not** the agent's submitted prompt — it is the commit finalizer's internal _follow-up_ prompt.

1. The TUI prompt panel resolves prompt content via `get_prompt_content()` → `AgentArtifactCache.select_prompt_file()`
   (`src/sase/agent/agent_artifacts_cache.py`). That method globs every `*_prompt.md` file in the agent's artifacts dir
   and selects the **most-recently-modified** one.

2. After the agent completes, the commit finalizer (`src/sase/llm_provider/commit_finalizer.py:212-216`) writes its
   follow-up prompt to `commit_finalizer_pass_{N}_prompt.md`. That filename **matches the `*_prompt.md` glob** and is
   newer than the original `…_prompt.md`, so it wins selection.

3. The follow-up prompt is built by `build_follow_up_prompt()`
   (`src/sase/llm_provider/commit_finalizer_prompting.py:124`) as:

   ```
   {original_prompt}

   --- Work So Far ---
   {accumulated_response}

   --- Commit Finalizer Pass {n} of {max} ---
   {details}
   ...
   ```

   `original_prompt` for a fork already ends with `# New Query\n{query}`, so once `--- Work So Far ---` and the
   finalizer details are appended, the New Query is no longer at the bottom.

**Deterministic reproduction** (already verified): an artifacts dir containing an original `sase_prompt.md` (ending in
`# New Query`) plus a newer `commit_finalizer_pass_1_prompt.md` causes `select_prompt_file()` to return the finalizer
file, whose content does **not** end with the New Query.

Note: `fork` is a _workflow_ (not an xprompt), so the internal `---` in fork.yml is **not** a multi-agent split
separator — that path was ruled out. fork.yml itself is correct; the bug is purely in which artifact the panel displays.

## Goal

The AGENT PROMPT panel should display the prompt that was actually **submitted to launch the agent** (original / step /
retry prompt) — not post-completion re-prompts generated by the commit finalizer. The finalizer's work remains visible
in the AGENT CHAT/REPLY section (it already has its own `…_response.md` files), so nothing is lost.

## Approach

Filter commit-finalizer follow-up prompt artifacts out of prompt-file _selection_, while leaving the files on disk (they
remain useful debugging artifacts and the existing test that asserts their existence keeps passing).

### Primary change — `src/sase/agent/agent_artifacts_cache.py`

- In `select_prompt_file()`, when building `candidates`, skip entries that are commit-finalizer follow-up prompts
  (filename matching `commit_finalizer_pass_*_prompt.md`). Keep the existing "most-recently-modified" behavior for the
  remaining (genuine) prompt files so retries still surface the latest submitted prompt.
- Implement the exclusion as a small, named predicate (e.g. `_is_finalizer_followup_prompt(name)`) rather than an inline
  regex, so intent is clear and the rule is reusable.

### Consistency change — `src/sase/core/agent_artifact_defaults.py`

- `_prompt_artifact_files()` (used by `_discover_prompt_image_paths`) globs the same `*_prompt.md` pattern. Apply the
  same exclusion there so finalizer follow-ups are not re-scanned for image paths (low impact, but keeps the "prompt
  artifact set" definition consistent in one place).

### Shared definition

- Define the filename rule once and reuse it in both call sites to avoid drift. Preferred home: a small helper/constant
  near where the finalizer _writes_ the file (e.g. alongside `commit_finalizer`/`commit_finalizer_prompting`), exported
  for the readers to import — so the writer and the display filter stay in sync. Final placement to be confirmed during
  implementation to respect existing module boundaries (the Rust-core boundary does not apply here; this is
  presentation-layer artifact selection).

## Out of Scope / Considered

- Changing `build_follow_up_prompt` ordering or re-emphasizing the New Query at the bottom of the finalizer prompt —
  rejected; the follow-up structure (task → work so far → finalizer instruction) is correct for what the LLM receives.
- Renaming `commit_finalizer_pass_{N}_prompt.md` to a non-`_prompt.md` name — workable alternative, but it changes a
  debugging artifact's name and the existing test assertion; the selection-layer filter is more targeted and lower-risk.
- `followup_prompt.md` (retry/exec-plan continuation) also ends with `_prompt.md`. It is a _legitimate_ re-prompt the
  agent received and is not part of this report, so it is intentionally left as-is. Flag for follow-up if it later
  proves to hijack the panel similarly.

## Verification

1. **Unit test** in `tests/agent/test_agent_artifacts_cache.py`: an artifacts dir with a `*_prompt.md` original and a
   newer `commit_finalizer_pass_1_prompt.md`; assert `select_prompt_file()` returns the original (not the finalizer
   file).
2. Confirm `tests/llm_provider/test_codex_fallback_invocation.py` still passes (the finalizer file is still written to
   disk).
3. Add/extend a prompt-panel rendering test (around `tests/ace/tui/widgets/test_prompt_artifact_cache.py` /
   `tests/test_ace_tui_widgets.py`) verifying the displayed AGENT PROMPT ends with the `# New Query` block when a
   finalizer pass prompt is also present.
4. `just check` (lint + mypy + tests). `just install` first per workspace rules.

## Risk

Low. The change only affects which on-disk prompt artifact the panel selects; no files are removed, no LLM-facing prompt
construction changes, and the finalizer's response content remains visible in the chat/reply section.
