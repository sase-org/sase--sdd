# Rename AMD Long-Term Memory Heading To Tier 2

## Goal

Change AMD-managed `AGENTS.md` documents so the long-term memory section is labeled:

```markdown
## Tier 2 (long-term) Memory
```

instead of:

```markdown
## Tier 3 (long-term) Memory
```

Then update `sase amd init` so regenerated documents use the new heading, run the command, and verify the generated
project and configured home/chezmoi `AGENTS.md` files are current.

## Current Understanding

The visible long-term memory heading is emitted by `render_managed_agents()` in `src/sase/amd/_memory.py`. The same
section is parsed by `src/sase/amd/_agents_doc.py`, where the structural parser currently recognizes
`## Tier 3 (long-term) Memory` and legacy `## Long-Term Memory Files` headings.

The command path is:

- `sase amd init`
- `src/sase/amd/_runner.py`
- `src/sase/amd/_planner.py`
- `render_managed_agents()`

Because `use_chezmoi: true` is configured for this machine, running `sase amd init` from this project plans both the
current project root and `/home/bryan/.local/share/chezmoi/home`. The chezmoi source `home/AGENTS.md` is already dirty
from the previous marker-removal work; this change should work with that file rather than reverting it.

There are only three checked-in `AGENTS.md` files in this repo, and only the root `AGENTS.md` currently contains the
Tier 3 long-memory heading. The other `AGENTS.md` files contain local instructions for tools and the ACE package and
should not be rewritten by this task unless they contain the old heading.

## Compatibility Contract

Newly rendered AMD-managed documents should use `## Tier 2 (long-term) Memory`.

The parser should still recognize `## Tier 3 (long-term) Memory` as an AMD long-memory section during migration. That
keeps `sase amd list`, description preservation, and drift detection useful for documents generated by older versions
until the next `sase amd init` rewrite updates them.

The parser should continue recognizing `## Long-Term Memory Files` as the older migration-only heading. The
`#### Long-Term Memory Files` list anchor should remain unchanged.

This task is only about the AMD memory tier label in `AGENTS.md`. It should not rename unrelated "Tier 2" terminology in
agent-loader tests or runtime code, where Tier 2 means source-scan reconciliation rather than long-term memory.

## Implementation Steps

1. Update AMD rendering in `src/sase/amd/_memory.py`.
   - Change the emitted long-term memory heading to `## Tier 2 (long-term) Memory`.
   - Leave the explanatory text and `#### Long-Term Memory Files` subheading unchanged.
   - Keep the explicit assertions that the old dynamic-memory heading is not rendered, but make sure they do not reject
     the new long-term Tier 2 heading.

2. Update structural parsing in `src/sase/amd/_agents_doc.py`.
   - Add `## Tier 2 (long-term) Memory` as the preferred/current long-section heading.
   - Keep `## Tier 3 (long-term) Memory` and `## Long-Term Memory Files` accepted for migration.
   - Avoid broad string matching that would accidentally classify `## Tier 2 (dynamic) Memory` as long-term memory.

3. Update focused tests.
   - `tests/main/test_amd_init.py`: generated documents should contain `## Tier 2 (long-term) Memory` and not contain
     `## Tier 3 (long-term) Memory` or the old dynamic-memory heading.
   - `tests/main/test_init_memory_handler.py`: AMD-enabled memory init should assert the new long-term heading.
   - `tests/main/test_amd_list.py`: the managed fixture should use the new heading, and at least one fixture should keep
     the old Tier 3 heading to prove migration parsing still works.

4. Regenerate generated `AGENTS.md` files by running `.venv/bin/sase amd init`.
   - Expected project diff: root `AGENTS.md` changes only the long-term memory heading from Tier 3 to Tier 2.
   - Expected chezmoi source diff: `/home/bryan/.local/share/chezmoi/home/AGENTS.md` should be regenerated with the new
     Tier 2 heading and the current marker-free renderer.
   - Do not edit memory files directly.

5. Run verification.
   - Focused pytest:

     ```bash
     .venv/bin/python -m pytest tests/main/test_amd_init.py tests/main/test_init_memory_handler.py tests/main/test_amd_list.py
     ```

   - Full repo check, as required after code changes:

     ```bash
     just check
     ```

   - Idempotence and sweep:

     ```bash
     .venv/bin/sase amd init
     rg -n "Tier 3 \\(long-term\\) Memory|## Tier 2 \\(dynamic\\) Memory|sase-amd" AGENTS.md src/sase/amd tests/main
     rg -n "Tier 3 \\(long-term\\) Memory" /home/bryan/.local/share/chezmoi/home/AGENTS.md
     git diff --check
     ```

## Risks And Guardrails

- The phrase "Tier 2" is used elsewhere in SASE for agent-loader reconciliation. Those references are unrelated and
  should stay as-is.
- The old `## Tier 2 (dynamic) Memory` heading was intentionally removed in earlier work. Tests should guard against
  reintroducing dynamic memory while still allowing `## Tier 2 (long-term) Memory`.
- Do not drop Tier 3 parser compatibility in the same change; otherwise existing AMD-managed documents would be
  misclassified until rewritten.
- Do not touch canonical memory files without explicit approval.
- Do not revert the pre-existing dirty chezmoi `home/AGENTS.md`; let `sase amd init` produce the expected generated
  content for this task.

## Acceptance Criteria

- New AMD-managed `AGENTS.md` output uses `## Tier 2 (long-term) Memory`.
- Old `## Tier 3 (long-term) Memory` documents still parse as managed long-memory documents.
- The repo root `AGENTS.md` is regenerated with the new heading.
- The configured chezmoi source `home/AGENTS.md` is regenerated by `sase amd init` with the new heading.
- Focused tests and `just check` pass.
- A final idempotence run of `sase amd init` reports that agent markdown documents are current.
