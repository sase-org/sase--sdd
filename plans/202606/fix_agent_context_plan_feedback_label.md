# Fix AGENT CONTEXT Labels for Plan Feedback Family Members

## Problem

The Ace Agents detail panel currently labels audited AGENT CONTEXT rows for a plan-feedback family member such as
`8q--plan-0` as `fb2`. In the screenshot, the `bob-cli` memory read at local `09:12:50` / UTC `2026-06-16T13:12:50`
should be attributed to the concrete family suffix `plan-0`, not the semantic feedback-round abbreviation `fb2`.

The underlying audit row is not the source of the bad label. The `bob-cli` memory log row records:

- `agent_name: "8q"` because the inherited root agent name is still in the environment.
- `artifacts_dir: "/home/bryan/.sase/projects/bob-cli/artifacts/ace-run/202606/16/20260616091220"`.

That artifact directory's `agent_meta.json` records:

- `name: "8q--plan-0"`
- `role_suffix: "--plan-0"`
- `agent_family: "8q"`
- `agent_family_role: "feedback"`

So the context loader is correctly matching the event to the family member by artifact directory. The incorrect part is
the compact display label generated by `src/sase/ace/tui/agent_context_members.py`: feedback suffixes are converted
through `plan_chain_feedback_round()` to labels like `fb2`.

## Desired Behavior

For family-context role labels in the MEMORY and SKILLS lanes:

- Plain plan member `--plan` keeps rendering as `plan`.
- Coder/question/epic/legend/commit members keep rendering as `coder`, `q`, `epic`, `legend`, and `commit`.
- Phase feedback members with suffixes like `--plan-0` and `--plan-1` render as `plan-0` and `plan-1`.
- Legacy numeric feedback members such as `.2`, `-2`, or `--2` can keep rendering as `fb2`, because they do not carry a
  phase-specific suffix name.
- Matching remains artifact-dir-first, with agent-name fallback only when the audit event lacks an artifact directory.

## Implementation Plan

1. Update `src/sase/ace/tui/agent_context_members.py`.
   - Add a small helper to convert a canonical plan-chain suffix into a compact suffix label by stripping the leading
     `--`.
   - In `_compact_role_label()`, handle canonical suffixes matching the phase-feedback shape `--plan-<token>` before
     calling `plan_chain_feedback_round()`.
   - Return `plan-<token>` for these members, so `--plan-0` renders as `plan-0`.
   - Leave legacy numeric feedback handling as `fbN`.
   - Update the docstring/comment that currently says all feedback rounds map to `fbN`.

2. Add focused regression coverage for MEMORY context labels.
   - Extend `tests/ace/tui/test_memory_reads_loader.py` with a family-context case where a root `alpha--plan` has a
     feedback follow-up `alpha--plan-0`.
   - Write one memory-read event against the feedback follow-up artifact directory.
   - Assert the returned `MemoryReadDisplayEvent.agent_label` is `plan-0`, not `fb2`.

3. Mirror the regression for SKILLS context labels.
   - The same helper drives `load_skill_uses_for_agent_context()`, so add the equivalent test in
     `tests/ace/tui/test_skill_uses_loader.py`.
   - This prevents the two AGENT CONTEXT lanes from diverging.

4. Verify existing behavior stays intact.
   - Run the targeted tests:
     `uv run pytest tests/ace/tui/test_memory_reads_loader.py tests/ace/tui/test_skill_uses_loader.py tests/test_plan_chain_roles.py`
   - If the helper change touches phase-label expectations indirectly, also run:
     `uv run pytest tests/ace/tui/widgets/test_agent_display_phase_labels.py`

## Risk and Scope

This should be a display-only change. It should not modify audit logging, memory-read validation, event persistence,
agent loading, or artifact matching. The only user-visible behavior change should be that AGENT CONTEXT role labels
prefer concrete phase suffix names for phase-feedback members such as `plan-0`.
