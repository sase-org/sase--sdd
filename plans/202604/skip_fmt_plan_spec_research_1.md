# Skip formatting plan/spec/research files in `just fmt`

## Problem

`just fmt-md` currently runs:

```
prettier --write --prose-wrap=always --print-width=120 "**/*.md"
```

This picks up every Markdown file in the repo, including three classes of files that the user does not want formatted:

1. **Checked-in planning artifacts** under the dedicated top-level directories:
   - `plans/` (e.g. `plans/202604/…md`)
   - `specs/` (e.g. `specs/202604/…md`)
   - `sdd/research/` (e.g. `sdd/research/202604/agents_md_token_optimization.md`)

2. **Ephemeral root-level plan files** matching `sase_plan_*.md` (already gitignored; there are ~190 of them in the repo
   root right now). These are generated by the `/sase_plan` skill and handed off to `sase plan`.

Reformatting these files is noisy (big diffs on every run), slow (~190+ files reflowed each invocation), and unwelcome —
the content is owned by planners/coders, not by the project's prose style.

The same glob is used by the CI-facing `just fmt-md-check` target, which runs in the `fmt-md-check` GitHub Actions job,
so CI also currently would flag these files if they drift from prettier style.

## Goal

When a user runs `just fmt` (or `just fmt-md` or `just fmt-md-check`), prettier should skip:

- Everything under `plans/`, `specs/`, and `sdd/research/`
- Every root-level `sase_plan_*.md`

All other Markdown (README, CLAUDE.md, AGENTS.md, `docs/**`, `src/**/*.md`, skill files, memory files, etc.) should
continue to be formatted exactly as it is today.

Explicitly out of scope:

- `docs/change_spec.md`, `docs/project_spec.md`, `docs/workflow_spec.md` — these are long-form design docs that happen
  to have "spec" in the name; they should stay formatted like the rest of `docs/`.

## Design

Use prettier's standard `.prettierignore` mechanism. Prettier picks it up automatically from the working directory, so a
single file change covers `fmt-md`, `fmt-md-check`, and any future prettier invocation (including CI, since the CI
`fmt-md-check` job runs the same `just` target).

Create `.prettierignore` at the repo root with:

```
# Planning artifacts — owned by planner/coder agents, not the project's prose style
plans/
specs/
sdd/research/
sase_plan_*.md
```

No change to the `Justfile` is required: the glob `**/*.md` stays as-is, and prettier filters the set.

### Why `.prettierignore` over modifying the glob

- **Single source of truth.** One file covers both `fmt-md` (write) and `fmt-md-check` (CI). If we instead edited the
  glob, we'd have to keep two invocations in sync and risk CI-vs-local drift.
- **Discoverable.** `.prettierignore` is the conventional place contributors look for "why isn't prettier touching this
  file?" — more discoverable than a custom shell-glob exclusion inside the `Justfile`.
- **Works with editor integrations.** VS Code / Neovim prettier plugins read `.prettierignore` too, so format-on-save in
  editors also stops touching these files. A Justfile-only change would not fix that.

### Verification

1. `just fmt-md` should complete without listing any file under `plans/`, `specs/`, `sdd/research/`, or any `sase_plan_*.md`
   in its "formatted" output.
2. `just fmt-md-check` (and by extension `just check`) should pass even if those files contain non-prettier-conformant
   Markdown.
3. `just fmt-md` still reformats a deliberately mis-wrapped line inside e.g. `README.md` or an
   `src/sase/xprompts/skills/*.md` file.

## Risks / notes

- CI: the `fmt-md-check` GitHub Actions job will automatically honor the new `.prettierignore` (it runs the same
  `just fmt-md-check` target). No workflow edits needed.
- Gitignore interaction: `sase_plan_*.md` files are already gitignored, but prettier doesn't read `.gitignore` by
  default, which is why they were still being rewritten. Listing them in `.prettierignore` is the correct fix.
- Prettier version: the repo installs prettier globally in CI (`npm install -g prettier`) and relies on whatever the
  contributor has locally. `.prettierignore` has been supported since prettier 1.x, so no version floor changes.
